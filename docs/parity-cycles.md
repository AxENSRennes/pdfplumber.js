# Parity Cycles

This repository includes parity-cycle corpora under `test/fixtures/parity-cycles`.
Synthetic diagnostic cycles are generated by `scripts/generate-parity-cycle-cases.py`.
Real-document cycles are generated by `scripts/generate-real-parity-cycle-cases.py`
from established external/upstream PDF corpora. Expected outputs are generated
automatically with the vendored Python `pdfplumber` oracle via `wsl_venv`.

## Cycle Layout

Each counted cycle contains 120 page cases:

- 100 working cases in `test/fixtures/parity-cycles/cycle-XX/working`
- 20 holdout cases in `test/fixtures/parity-cycles/cycle-XX/holdout`

Synthetic cycles use generated micro-PDF diagnostics. Real-document cycles use
selected pages from PDFs listed in `test/fixtures/external-pdfs/manifest.json` or
`test/fixtures/external-holdout-pdfs/manifest.json`; generated PDFs are not counted
in those real-document cycles. The tracked manifests record each PDF path, source
manifest, source URL, URL origin, source corpus, terms, metadata, category, size,
selected pages, rationale, and SHA-256 hash. The tracked goldens live in
`test/fixtures/goldens/parity-cycles`.

## Commands

Regenerate the synthetic diagnostic PDFs, manifests, and Python oracle goldens:

```sh
npm run fixtures:generate:parity-cycles
```

Regenerate the current real-document streak and its shard tests:

```sh
npm run fixtures:generate:real-parity-cycles
npm run fixtures:generate:real-parity-test-shards
```

Run only the working corpus:

```sh
npm run test:cycle-working-parity
```

Run only the holdout corpus:

```sh
npm run test:cycle-holdout-parity
```

Those aggregate commands default to the current real-document streak,
cycles 12, 13, and 14. The holdout command enables the classified-miss guard used
for cycle 14.

Run selected cycles:

```sh
PDFPLUMBER_JS_CYCLES=14 npm run test:cycle-working-parity
PDFPLUMBER_JS_CYCLES=14 npm run test:cycle-holdout-parity
```

Run the real-document shard gate with classified holdout handling:

```sh
node scripts/run-real-cycle-shards.mjs --phase working --cycles 12,13,14
node scripts/run-real-cycle-shards.mjs --phase holdout --cycles 12,13,14
```

Raw holdout shard commands intentionally report exact mismatches. The real-document
runner accepts only classified holdout misses recorded in
`test/fixtures/parity-cycles/classified-holdout-failures.json`, and refuses to
accept more than one classified holdout failure in any cycle. A classified miss
that starts passing must be removed from the classification file; the shard runner
fails stale classifications so the reported holdout count remains current.

## Notes

The parity-cycle suite is separate from the older in-repo and external parity suites.
As of this cycle addition, those older suites still expose unrelated pre-existing
failures in fonts, colors, curves, images, and complex CMaps. The generated cycle
suite is intended as a fast, oracle-backed regression gate for browser-capable core
behavior while broader real-world parity work continues.
