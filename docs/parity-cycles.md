# Parity Cycles

This repository includes a generated parity-cycle corpus under `test/fixtures/parity-cycles`.
The corpus is generated by `scripts/generate-parity-cycle-cases.py` and the expected
outputs are generated automatically with the vendored Python `pdfplumber` oracle via
`wsl_venv`.

## Cycle Layout

Each cycle contains 120 new micro-PDF cases:

- 100 working cases in `test/fixtures/parity-cycles/cycle-XX/working`
- 20 holdout cases in `test/fixtures/parity-cycles/cycle-XX/holdout`

The tracked manifests record each PDF path, generator provenance, category, size, and
SHA-256 hash. The tracked goldens live in `test/fixtures/goldens/parity-cycles`.

## Commands

Regenerate the PDFs, manifests, and Python oracle goldens:

```sh
npm run fixtures:generate:parity-cycles
```

Run only the working corpus:

```sh
npm run test:cycle-working-parity
```

Run only the holdout corpus:

```sh
npm run test:cycle-holdout-parity
```

Run selected cycles:

```sh
PDFPLUMBER_JS_CYCLES=1 npm run test:cycle-working-parity
PDFPLUMBER_JS_CYCLES=1 npm run test:cycle-holdout-parity
```

## Notes

The parity-cycle suite is separate from the older in-repo and external parity suites.
As of this cycle addition, those older suites still expose unrelated pre-existing
failures in fonts, colors, curves, images, and complex CMaps. The generated cycle
suite is intended as a fast, oracle-backed regression gate for browser-capable core
behavior while broader real-world parity work continues.
